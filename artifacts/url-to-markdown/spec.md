# spec.md — url-to-markdown

## 개요

웹 페이지 URL을 붙여넣으면 본문을 Markdown으로 추출해 보여주고, 복사·다운로드·LLM 서비스로 바로 넘길 수 있는 단일 페이지 도구.

## 범위

### 포함
- URL 입력 및 지우기
- URL → Markdown 변환 (defuddle 기반)
- 결과 확인 (제목·저자 헤더 + 렌더링된 Markdown)
- Markdown 복사 / .md 파일 다운로드
- ChatGPT·Claude로 열기 (새 탭 + 프롬프트+Markdown 클립보드 복사)
- 프롬프트 프리셋 3종 및 직접 입력 (1회용)
- 다크모드 토글

### 제외
- 변환 결과 저장·히스토리 — 이번 범위 밖. 세션 내 1회용으로 충분
- 로그인/인증 — 공개 도구, 사용자 계정 불필요
- 모바일 특화 레이아웃 — plan.md에서 반응형 여부 결정

## 시나리오

### 1. URL 입력 후 변환 성공

- **Given** — URL 입력 필드가 비어 있는 초기 화면
- **When** — 유효한 URL을 붙여넣고 "변환" 버튼 클릭
- **Then** — 로딩 인디케이터가 표시되다가, 완료 후 페이지 제목·저자 헤더와 렌더링된 Markdown이 나타난다

성공 기준:
- [ ] 변환 중 로딩 인디케이터가 화면에 보인다
- [ ] 완료 후 추출된 페이지 제목이 헤더 영역에 표시된다
- [ ] 완료 후 저자 정보가 있으면 헤더 영역에 표시된다
- [ ] 완료 후 본문이 렌더링된 Markdown으로 결과 영역에 표시된다

### 2. 지우기 버튼으로 리셋

- **Given** — URL 입력 필드에 값이 있는 상태 (결과가 표시된 상태 포함)
- **When** — 지우기 버튼 클릭
- **Then** — URL 입력 필드가 비워지고, 결과 영역이 사라지며 초기 화면으로 돌아간다

성공 기준:
- [ ] 지우기 버튼 클릭 후 URL 입력 필드가 빈 문자열이 된다
- [ ] 지우기 버튼 클릭 후 결과 영역(제목·저자·Markdown)이 화면에서 사라진다

### 3. 변환 실패 — 잘못된 URL

- **Given** — URL 입력 필드에 값이 있는 상태
- **When** — 유효하지 않은 형식의 URL 또는 접근 불가 URL을 입력하고 "변환" 버튼 클릭
- **Then** — 결과 영역 대신 URL 입력 필드 아래(또는 결과 영역)에 에러 메시지가 표시된다. 입력 필드는 그대로 유지된다

성공 기준:
- [ ] 유효하지 않은 URL 입력 시 에러 메시지가 인라인으로 나타난다
- [ ] 에러 메시지 표시 후 URL 입력 필드의 값이 유지된다
- [ ] 에러 메시지 표시 후 로딩 인디케이터가 사라진다

### 4. Markdown 복사하기

- **Given** — 변환 결과가 표시된 상태
- **When** — "복사하기" 버튼 클릭
- **Then** — Markdown 전문이 클립보드에 복사되고, 버튼 또는 근처에 복사 완료 피드백이 잠시 표시된다

성공 기준:
- [ ] "복사하기" 클릭 후 클립보드에 Markdown 텍스트가 담긴다
- [ ] "복사하기" 클릭 후 완료 피드백(예: "복사됨")이 화면에 나타난다

### 5. .md 파일 다운로드

- **Given** — 변환 결과가 표시된 상태
- **When** — "다운로드" 버튼 클릭
- **Then** — Markdown 전문이 담긴 `.md` 파일이 브라우저 다운로드로 저장된다

성공 기준:
- [ ] "다운로드" 클릭 후 `.md` 확장자 파일 다운로드가 시작된다
- [ ] 다운로드된 파일에 추출된 Markdown 전문이 포함된다

### 6. ChatGPT / Claude로 열기

- **Given** — 변환 결과가 표시된 상태, 프롬프트는 선택됐거나 선택되지 않은 상태 모두 포함
- **When** — "ChatGPT로 열기" 또는 "Claude로 열기" 버튼 클릭
- **Then** — 해당 서비스가 새 탭에서 열리고, "(선택된 프롬프트 +) Markdown 전문"이 클립보드에 복사된다. 클립보드 복사 완료 피드백이 표시된다

성공 기준:
- [ ] 버튼 클릭 시 ChatGPT 또는 Claude가 새 탭에서 열린다
- [ ] 프롬프트가 선택된 경우 "프롬프트\n\nMarkdown 전문" 형태로 클립보드에 복사된다
- [ ] 프롬프트가 없는 경우 Markdown 전문만 클립보드에 복사된다
- [ ] 클립보드 복사 완료 피드백이 화면에 나타난다

### 7. 프롬프트 프리셋 선택

- **Given** — 변환 결과가 표시된 상태, 프롬프트 선택 UI가 보이는 상태
- **When** — "요약해줘" / "한국어로 번역해줘" / "쉽게 설명해줘" 중 하나 선택
- **Then** — 선택한 프리셋이 활성화 표시되고, 이후 LLM으로 열기 시 해당 프롬프트가 앞에 붙는다

성공 기준:
- [ ] 프리셋 선택 후 해당 항목이 선택됨 상태로 표시된다
- [ ] 다른 프리셋을 선택하면 이전 선택이 해제되고 새 프리셋만 활성화된다

### 8. 직접 입력 프롬프트

- **Given** — 프롬프트 선택 UI에서 "직접 입력" 선택
- **When** — 텍스트 입력 필드가 노출되면 원하는 프롬프트를 입력
- **Then** — 입력한 텍스트가 LLM으로 열기 시 프롬프트로 사용된다. 페이지를 벗어나거나 리셋하면 입력값이 사라진다 (저장하지 않음)

성공 기준:
- [ ] "직접 입력" 선택 시 텍스트 입력 필드가 나타난다
- [ ] 프리셋 선택으로 전환하면 직접 입력 필드가 사라진다
- [ ] 지우기 버튼으로 리셋하면 직접 입력값이 초기화된다

### 9. 다크모드 토글

- **Given** — 화면에 다크모드 토글 버튼이 보이는 상태
- **When** — 토글 버튼 클릭
- **Then** — 라이트/다크 테마가 전환된다. 새로고침 후에도 마지막 선택이 유지된다

성공 기준:
- [ ] 토글 클릭 시 화면 배경·텍스트 색상이 다크/라이트로 전환된다
- [ ] 페이지 새로고침 후 마지막으로 선택한 테마가 유지된다

## 불변 규칙

**프롬프트 미선택 시 기본값**: 프롬프트를 아무것도 선택하지 않은 상태에서 "복사하기" / LLM 열기 시, Markdown만 전달된다. 프롬프트를 강제하지 않는다.

## 의존성
